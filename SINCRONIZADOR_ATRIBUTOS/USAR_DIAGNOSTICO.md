# ?? Herramienta de Diagnóstico de Atributos

## ?? Problema Actual

El sincronizador **NO está leyendo** los atributos `ESTATUS_PIEZA` y `PRIORIDAD` de la Main Part, aunque **SÍ tienen valores** (`CONECTADO` y `1`).

**Causa probable**: El nombre del atributo en Tekla es **diferente** al que estamos buscando.

---

## ? Solución: Usar la Herramienta de Diagnóstico

He agregado un **botón de diagnóstico** (??) que te mostrará el **nombre exacto** de todos los atributos del objeto seleccionado.

### ?? Cómo Usar

1. **Ejecuta el sincronizador**:
   ```cmd
   cd SINCRONIZADOR_ATRIBUTOS
   ejecutar.bat
   ```

2. **Haz clic en el botón** `?? Diagnóstico de Atributos`

3. **Selecciona la Main Part** en Tekla (la que tiene `CONECTADO` y `1`)

4. **Presiona ENTER**

5. **Verás un reporte** con todos los atributos encontrados

## ?? Siguiente Paso

### Si el diagnóstico encuentra los atributos

**Perfecto!** Copia el **nombre exacto** que aparece en el reporte y compártelo conmigo. Por ejemplo:

```
Encontré estos nombres:
- [ESTADO_FABRICACION] = 'CONECTADO'
- [PRIORIDAD_DET] = '1'
```

Entonces actualizaré el código para usar esos nombres exactos.

### Si el diagnóstico NO encuentra los atributos

Necesito que hagas esto en Tekla:

1. **Selecciona la Main Part**
2. **Doble clic** para abrir propiedades
3. **Busca** dónde aparece "CONECTADO" y "1"
4. **Copia el nombre del campo** (no el valor, sino el label/título del campo)
5. **Compártelo** conmigo

Por ejemplo:
- En la pestaña "User-defined attributes"
- Campo: `Status de Fabricación:` ? este es el nombre
- Valor: `CONECTADO` ? este es el valor
